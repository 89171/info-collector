# 实时信息收集助手 (Info Collector)

一个专业的实时信息收集技能模块，擅长从多源公开渠道快速聚合特定主题的最新资讯，并进行结构化输出。

## 功能概述

该技能模块可以：
- 🔍 **多源信息检索** - 从 RSS、新闻网站、社交媒体、代码仓库、论坛等渠道检索信息
- ✅ **真实性校验** - 对所有信息进行多维度真实性验证和交叉验证
- 🧹 **智能去重合并** - 去重和合并相同事件的多个报道
- 📊 **结构化分类** - 按官方动态、媒体报道、社区讨论等七大类别分类
- 🎯 **优先级排序** - 按时效性、可信度、相关性等多维度排序
- 📄 **报告生成** - 输出格式统一、内容完整的结构化报告

## 核心能力

### 1. 多源信息检索
覆盖以下渠道（按优先级）：
- **官方渠道** - 官网、官方博客、GitHub官方仓库、认证账号
- **权威媒体** - 科技媒体、行业媒体、主流新闻网站
- **开发者生态** - GitHub Release/Issue/PR、技术论坛
- **社交平台** - X/Twitter、微博、微信公众号
- **社区讨论** - Reddit、Hacker News、V2EX、贴吧等
- **搜索引擎** - 综合检索补充

### 2. 真实性校验
每条信息必须通过以下检查：
- ✓ 存在明确可访问的原始链接
- ✓ 有可识别的发布时间戳
- ✓ 来源可追溯（官方/认证账号/权威媒体）
- ✓ 内容可被独立交叉验证
- ✓ 非明显广告软文或未经证实的爆料

### 3. 分类体系
- **官方动态** - 官方公告、更新、发布
- **媒体报道** - 深度报道、分析、新闻
- **社区讨论** - 技术论坛、热帖、讨论
- **社交热议** - 社交平台关键话题
- **开发者生态** - Release、Issue、PR、文档
- **深度解读** - 分析、观点、研究报告
- **市场信号** - 融资、合作、产品动态

## 使用方法

### 基本调用

```python
# 标准收集指令
请围绕主题"{topic}"执行实时信息收集：
- 时间窗口：48小时（默认）
- 语言：zh-CN（默认）
- 每类上限：8条（默认）
- 严格时效：true（默认）
```

### 输入参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `topic` | string | 必需 | 收集的主题（产品名、技术名、事件名等） |
| `time_window_hours` | int | 48 | 时间窗口（单位：小时） |
| `language` | string | zh-CN | 输出语言（zh-CN/en-US） |
| `regions` | list | [] | 区域偏好 |
| `must_include_sources` | list | [] | 必须覆盖的渠道 |
| `exclude_sources` | list | [] | 排除的渠道 |
| `max_items_per_category` | int | 8 | 每个分类的最大条目数 |
| `strict_recency` | bool | true | 是否严格限制时间窗口 |

## 执行流程

1. **主题解析** - 提取关键词、同义词、缩写，生成中英文检索组合
2. **并行检索** - 同时从多个渠道检索信息
3. **真实性校验** - 验证每条信息的可靠性
4. **去重合并** - 识别和合并重复信息
5. **分类组织** - 按七大类别分类各条信息
6. **优先排序** - 按时效性、可信度、相关性排序
7. **报告生成** - 输出结构化报告

## 推荐使用场景

## 使用建议

1. **明确主题** - 提供具体的产品名、技术名或事件名
2. **合理时间窗**
   - 48小时 - 适合热点追踪
   - 7天 - 适合趋势观察
   - 30天 - 适合长期监控
3. **指定渠道** - 如需覆盖特定渠道，可明确说明
4. **反馈修正** - 发现信息缺失或偏差时，调整参数重新检索
5. **人工复核** - 对重要决策建议人工复核关键信息来源

## 限制说明

本模块存在以下限制：
- ❌ 无法访问需要登录或付费的内容
- ❌ 无法抓取私密/非公开社交媒体内容
- ❌ 部分已删除或不可访问的内容无法获取
- ❌ 秒级实时更新受检索频率限制
- ⚠️   微信等部分渠道可能因访问限制而缺失
- ⚠️   部分跨时区内容的时间判断可能有偏差

## 质量保证

- 优先质量而非覆盖率
- 仅保留可验证来源的信息
- 无确切发布时间的内容降级处理
- 未经证实的爆料标注"待验证"
- 摘要忠于原文，不夸大不编造
